Design System
Single source of truth for BklynClay Glaze Studio. Every token, component, and interaction pattern lives here. AI contributors must read the guardrails before writing any UI code.
Color Tokens

All colors are defined as CSS custom properties on :root. Never use raw hex values in component code — always reference a token. The only exception is one-off SVG fills inside JS-generated tile/swatch functions.

Spacing Scale

BklynClay uses a loose 4px base grid. Use these values — don't introduce arbitrary px sizes. Gap between related items is always smaller than gap between sections.

2pxGap between swatches, dots
3pxGap between glaze chips
4pxInline badge padding, dense button xs
5pxCard button gap, artist chip grid gap

Rule: Padding inside a component uses the same value on all axes (e.g. padding:6px 6px) or smaller vertical than horizontal (e.g. 5px 7px). Never more vertical than horizontal.

Iconography

BklynClay uses Unicode text characters and symbols only — no icon library, no SVG icon sprites, no emoji. All icons are inline text glyphs rendered at the font-size of their containing element.

Usage Rules

✓ Do
  • Use .primary for the single most important action per surface
  • Use .btn.sm inside card footers
  • Use .btn.xs for paired action buttons in tight spaces
  • Use .ghost for low-priority icon-adjacent actions
  • Use .remove-subtle for destructive remove actions
  • Use .shuffle-btn only for the full-width primary CTA at panel bottom
✗ Don't
  • Don't create new button classes — use existing variants
  • Don't place two .primary buttons side by side
  • Don't use .pin-on for anything other than save/pin state
  • Don't add icons inside .btn — text only
  • Don't change border-radius — always inherits --r
  • Don't use <div> or <a> as buttons

Transitions

All buttons use transition: background .12s, color .12s, border-color .12s. Never change this duration or add other properties to the transition.

External JS libraryNeverNo React, Vue, Alpine, jQuery. Plain vanilla JS only.
Icon libraryNeverNo Heroicons, Lucide, FontAwesome. Use Unicode text glyphs (see Iconography).
Emoji as iconsNeverEmoji renders inconsistently. Only existing 🖼 in drop zone is permitted.
rem / em unitsNeverAll sizes in px only. Body is 14px. Font scale is fixed — do not make it fluid.
Raw hex in componentsAvoidUse CSS custom properties. Exception: SVG fill values inside JS tile functions.
New custom propertiesAvoidUse existing tokens. If you must add one, declare it on :root and document it here.
Inline style for layoutAvoidUse classes. Inline style only for dynamic values (e.g. width from JS %).
border-radius other than documentedAvoidUse --r (6px) by default. Only use documented exceptions (2,3,4,5,8,10,20px).
box-shadow for cardsAvoidCards use border only. Box-shadow is only for: modals, floating bars, selected states, hover on rank options.
Multiple primary buttonsNeverOne .primary per surface. Use .btn (default) for all secondary actions.
innerHTML with user dataNeverXSS risk. Use textContent or DOM methods for any user-supplied strings.
New font or typefaceNeverSystem font stack only (--font). No Google Fonts, no @font-face.
Animations on width/heightNeverOnly transition background, color, border-color, opacity, transform.

Patterns Reference

Active / Selected State Pattern

Every interactive component uses one of these three active state patterns. Match the component type:
  • Filled dark: background:var(--ink); color:#fff; border-color:var(--ink) — buttons, tabs, project tabs, chips
  • Accent border: border-color:var(--acc); box-shadow:0 0 0 2px var(--ink) — card selection
  • Accent color: color:var(--acc) — pin icons, paired item marks

Hover State Pattern

  • Buttons / chips: background:var(--surf); border-color:var(--border2); color:var(--ink)
  • List rows: background:var(--surf) only
  • Remove / danger: border-color:#c00; color:#c00; background:#fff5f5
  • Ghost buttons: background:var(--surf); border-color:var(--border)

Drag State Pattern

  • Dragging item: opacity:.4
  • Drop target card: border-color:var(--acc); box-shadow:0 0 0 2px rgba(184,69,16,.22)
  • Drop target chip: border-color:var(--acc); background:#fff8f5
  • Drop target project tab: background:#dff0e8; outline:2px dashed #5aaa7a

Adding a New Component — Checklist

Before writing CSS, verify:
  1. Does an existing component cover this need? (check this document first)
  2. Are you using only tokens from the color section above?
  3. Border is 1px (divider) or 1.5px (interactive) — nothing else?
  4. Border-radius uses var(--r) or a documented exception?
  5. Font sizes are from the type scale (9/10/11/12/13/14/15px)?
  6. Spacing values are from the spacing scale above?
  7. Hover/active states follow the three patterns documented above?
  8. Transition only uses approved properties and durations?
  9. No external libraries, fonts, or icon sets introduced?
